RHSPlanet3DPFFrame:

Path: Missions/LunarLanding

% Cartesian orbital equations in a planet fixed (rotating) frame.
 The gravity function returns the gravitational accelerations in the
 planet fixed frame. It must be of the form:

  a = GravFun( r, gFunData )
 
 where a is in the planet fixed frame. d.mu is not used in this case. The
 planet rotation is around the ECI z axis. The default data structure is for
 the the moon.
--------------------------------------------------------------------------
   Form:
   xDot = RHSPlanet3DPFFrame( x, ~, d )
--------------------------------------------------------------------------

   ------
   Inputs
   ------
   x     (7,1)   State [r;v;m]
   t     (1,1)   Time (not used)
   d     (.)     Data structure
                 .mu       (1,1) Gravitational constant
                 .gFun     (2,1) Pointer to the gravity function
                 .gFunData	(1,1) Gravity function data
                 .omega    (1,1) Planet rotation rate
                 .uThrust  (3,1) Unit thrust vector
                 .uE       (1,1) Exhaust velocity (m/s)
                 .thrust   (1,1) Thrust (N)

   -------
   Outputs
   -------
   xDot	(7,1)   State derivative [r;v;m]

--------------------------------------------------------------------------
